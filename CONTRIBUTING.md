# Contributing to BOSC Community Library

Thank you for your interest in contributing! Please follow these guidelines.

## Getting Started

### 1. Fork & Clone
```bash
git clone https://github.com/YOUR-USERNAME/BOSC-Community-Library.git
cd BOSC-Community-Library
```

### 2. Create a Branch
Always work on a new branch — never commit directly to `main`.
```bash
git checkout -b feature/your-feature-name
```

## How to Contribute

### Reporting Bugs
- Search existing issues first to avoid duplicates.
- Use the **Bug Report** issue template.
- Include clear steps to reproduce the problem.

### Suggesting Features
- Open an issue with the label `enhancement`.
- Describe the problem it solves and how it should work.

### Submitting a Pull Request
1. Ensure your branch is up to date with `main`.
2. Write clear, commented code.
3. Update documentation if needed.
4. Fill out the PR template completely.
5. Link the related issue using `Closes #issue-number`.

## Code Standards
- Use descriptive variable and function names.
- Follow consistent indentation (2 spaces for JS, 4 for Python).
- Write meaningful commit messages (e.g., `fix: correct typo in README`).

## Review Process
All PRs require at least one maintainer review before merging.
Feedback will be provided within 5 business days.
